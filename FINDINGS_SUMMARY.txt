SECURITY ISSUES (3 CRITICAL)
================================================================================

1. SHELL INJECTION via exec()
   - Arbitrary command execution with full privileges
   - No sandboxing or validation
   - Severity: CRITICAL
   - FIX: Remove or redesign with proper escaping

2. FILE ACCESS VULNERABILITY
   - Read any file on system
   - Write/overwrite any file
   - No path restrictions
   - Severity: CRITICAL
   - FIX: Implement whitelist, restrict to CWD

3. RESOURCE EXHAUSTION
   - No recursion depth limits
   - No memory limits
   - No execution timeout
   - Can crash host system
   - Severity: CRITICAL
   - FIX: Add resource limits and timeouts

EDGE CASES & BUGS (10 IDENTIFIED)
================================================================================

1. Negative array indexing wraps (arr[-1] doesn't work as expected)
2. Empty arrays with min()/max() return inconsistent values
3. Power operator with edge cases (0**-1, 0**0)
4. String-to-number conversion silently fails
5. Null field access returns null instead of error
6. Single-quoted strings don't support \n escapes
7. Floating point comparison (0.1 + 0.2 != 0.3)
8. Split with empty separator behavior undefined
9. Deep recursion causes stack overflow (no limit)
10. Function recursion broken in closures

================================================================================
PERFORMANCE ISSUES (3 IDENTIFIED)
================================================================================

1. Deep cloning on every function call (all variable bindings copied)
   - Solution: Use Rc<RefCell<>> or similar

2. String concatenation is O(n²) in loops
   - Solution: String builder or rope structure

3. Array operations not optimized (linear search, cloning)
   - Solution: Use iterators, COW semantics
